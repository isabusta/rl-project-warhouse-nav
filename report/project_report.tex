\section{Introduction}
Warehouse automation is an increasingly active area of research and industrial development, with a growing demand for fast, accurate and efficient order fufilment. Inefficient routing increases operational costs and reduces throughput. As warehouses grow in size, finding efficient strategies manually becomes impractical.

We consider a warehouse robot that must pick up packages and deliver them to storage locations in the right order, while avoiding obstacles. This problem is modeled as a Markov Decision Process (MDP) \cite{sutton2018}. The state variable captures the robot's position, the remaining packages, and the locations of obstacles. Possible actions include movement steps and package pickup or delivering packages. The reward system incentivises fast and successful deliveries, while penalising collisions with walls. We implement and evaluate five approaches on a discrete grid warehouse: Q-Learning, SARSA, Value Iteration, Policy Iteration, and Deep Q-Network (DQN) \cite{mnih2015}.
\subsection*{Related Work}
RL has been applied to warehouse robotics across a range of problem settings and scales \cite{punukollu2019integrating, peyas2021warehouse, liu2025drmarl}. A common observation is that warehouse environments are well-suited for RL because they are dynamic and unpredictable, which renders hand-crafted rules insufficient \cite{peyas2021warehouse}.

At the algorithm level, prior work has explored DQN, PPO, and Advantage Actor-Critic for tasks such as route planning, object retrieval, and resource allocation \cite{punukollu2019integrating}. Peyas et al.\ \cite{peyas2021warehouse} use both Deep RL for navigation and Q-learning for product placement in a 2D grid, which is the same action space and environment structure we adopt. At industrial scale, Liu et al.\ \cite{liu2025drmarl} apply multi-agent RL to package sorting in Amazon warehouses and achieve an 80\% reduction in package recirculation under uncertain conditions.

Two challenges appear across this work. Agents require many training episodes before convergence \cite{punukollu2019integrating}. Second, state spaces grow quickly as the problem scales: Peyas et al.\ \cite{peyas2021warehouse} acknowledge that extension to higher-dimensional, real-world environments remains unsolved. In our setting, each additional package doubles the number of states to experiment with higher-dimensional environments. We also test robustness by the addition of dynamic obstacles during execution.

\section{Preliminaries}

\subsection{MDP Formulation}

The warehouse problem is formalised as a tuple $(\states, \actions, \transition, \reward, \discount)$.

\paragraph{State space $\states$.}
Each state is a tuple $s = (\text{pos},\, c,\, \mathbf{d})$, where:
\begin{itemize}
    \item $\text{pos} = (r, c) \in \mathcal{G}$ is the robot's position, with $\mathcal{G}$ the set of passable cells in the grid,
    \item $c \in \{\emptyset, 0, 1, \ldots, N{-}1\}$ is the package currently carried ($\emptyset$ if none),
    \item $\mathbf{d} \in \{0,1\}^N$ is a binary vector indicating which of the $N$ packages have been delivered.
\end{itemize}
States where a delivered package is still being carried are excluded. The total number of states grows as $|\mathcal{G}| \times (N+1) \times 2^N$.

\paragraph{Action space $\actions$.}
There are six actions: $\actions = \{\texttt{up},\, \texttt{down},\, \texttt{left},\, \texttt{right},\, \texttt{pickup},\, \texttt{deliver}\}$. Movement actions are always available; \texttt{pickup} is valid only when the robot stands on an undelivered package and carries nothing; \texttt{deliver} is valid only when the robot carries a package and stands on its designated storage cell.

\paragraph{Transition function $\transition$.}
The environment is deterministic: $\transition(s' \mid s, a) \in \{0, 1\}$ for all $s, a, s'$. Movement to a wall leaves the robot in place. Pickup and deliver actions update $c$ and $\mathbf{d}$ accordingly.

\paragraph{Reward function $\reward$.}
The reward $\reward(s, a)$ is defined as:
\begin{equation}
    \reward(s, a) =
    \begin{cases}
        -1   & \text{valid movement step} \\
        -5   & \text{movement into a wall} \\
        -10  & \text{invalid pickup or deliver} \\
        +9   & \text{successful pickup } (+10 - 1) \\
        +99  & \text{successful delivery } (+100 - 1)
    \end{cases}
\end{equation}

\paragraph{Discount factor $\discount$.}
We use $\discount = 0.95$, which encourages the agent to complete deliveries quickly.

\paragraph{Bellman optimality equation.}
The optimal state-value function $\optsval$ satisfies:
\begin{equation}
    \optsval(s) = \max_{a \in \actions} \left[ \reward(s,a) + \discount \sum_{s' \in \states} \transition(s' \mid s, a)\, \optsval(s') \right]
\end{equation}

\paragraph{Optimal policy.}
The optimal policy selects the action that maximises the expected return in every state:
\begin{equation}
    \policy^*(s) = \arg\max_{a \in \actions} \left[ \reward(s,a) + \discount \sum_{s' \in \states} \transition(s' \mid s, a)\, \optsval(s') \right]
\end{equation}

\subsection{Deep Q-Network (DQN)}

We implemented a Deep Q-Network with a hidden dimension of 256 to approximate the Q-values. 
The network consists of three linear layers.
Transitions for updating the network are sampled from a replay buffer.
Actions are selected according to the $\varepsilon$-greedy principle: with probability 
$\varepsilon$ a random action is chosen (exploration), and with probability $1-\varepsilon$ 
the action with the highest Q-value is selected (exploitation). $\varepsilon$ starts at $0.9$ 
and is exponentially decayed to $0.01$ during training. The target network is updated via 
soft update with $\tau = 0.001$. 
The learning rate is initialized with $0.001$ and reduced by a factor of $\gamma$ using a MulitStepRL scheduler. As a basis for the implementation we used Lab 8 and the 
PyTorch documentation.
Detailed number in table \ref{tab:dqn-params}

\paragraph{State Encoding}
To feed the states to the network, we experimented with two different state encodings.

\subparagraph{Initial Encoding}
The state was represented as a flat vector:
\begin{equation}
    s = (\textit{robot\_pos},\; \textit{carrying},\; \textit{delivered})
\end{equation}
where $\textit{robot\_pos} = (row, col)$, $\textit{carrying} = i$ if the robot carries 
package $i$ (and $\textit{None}$ otherwise), and $\textit{delivered}_i = 1$ if package $i$ 
has been delivered. The carrying state was one-hot encoded and the complete state was passed 
to the network as a flattened vector. This encoding resulted in strictly negative rewards 
throughout training.

\subparagraph{Grid-based Encoding}
We switched to a multi-channel grid representation. Given $N$ packages, the state is encoded 
as $(2 + 3N)$ channels of size $rows \times cols$:
\begin{itemize}
    \item \textbf{Channel 0:} obstacle map — $1$ if a wall occupies position $(r, c)$
    \item \textbf{Channels $1$\,..\,$N$:} package positions — $1$ at the start position of 
          package $i$; set to $0$ once picked up or delivered
    \item \textbf{Channels $N{+}1$\,..\,$2N$:} storage locations — $0.5$ if empty, $1.0$ 
          if delivered
    \item \textbf{Channel $2N{+}1$:} robot position — $1$ at the current position of the robot
    \item \textbf{Channels $2N{+}2$\,..\,$3N{+}1$:} carrying state — channel $2N{+}2{+}i$ 
          is set to $1$ everywhere if the robot is carrying package $i$
\end{itemize}
All channels are concatenated and flattened into a vector before being passed to the network. 
Results remained negative.

\section{Experiments}

This section presents the conducted experiments.

Next, the experimental setups are described in detail, followed by a presentation of the results and a final discussion.

In this project, the following Reinforcement Learning algorithms are applied to the environment described above:

\begin{itemize}
    \item Q-Learning
    \item SARSA
    \item Value Iteration
    \item Policy Iteration
    \item Deep Q-Learning (DQN)
\end{itemize}

The Experiments for Value Iteration and Policy Iteration will be provided in the final version of this paper. They are not present in the current draft paper. 

\subsection{Hypotheses}
% List the specific questions the experiments aim to answer.

The goal of the experiments is to compare the performance and learning behavior of different Reinforcement Learning algorithms in the warehouse environment described in section \ref{sec:mdp}, MDP. 
We investigate differences between model-based and model-free approaches as well as tabular and deep learning methods.

We formulate the following hypotheses:

\begin{itemize}
    \item Model-based algorithms, such as Value Iteration and Policy Iteration, converge faster since they exploit full knowledge of the environment dynamics.
    
    \item Model-free algorithms, namely Q-Learning and SARSA, require more interaction with the environment but are able to learn effective policies without access to the transition model.
    
    \item Q-Learning is expected to achieve higher final performance than SARSA due to its off-policy learning strategy.

    \item SARSA is expected to required more iterations to find a optimal policy
    
    \item Deep Q-Learning is expected to scale better to larger state spaces but requires more training episodes to achieve stable performance.

    \item Higher learning rates ($\alpha$) lead to faster initial learning but may result in unstable convergence, whereas lower values produce more stable but slower learning behavior.

    \item Higher exploration rates ($\epsilon$) improve state-space exploration but slow convergence, while lower values may lead to suboptimal policies due to insufficient exploration. In this work, the exploration rate decays over time and is bounded by a minimum value of $\epsilon_{\min} = 0.01$, ensuring a small amount of continued exploration even in later training stages.
\end{itemize}

\subsection{Setup}
In order to conduct different experiments in a flexible manner, a graphical MDP configurator was implemented. 
It allows the configuration of both the environment and the hyperparameters of the reinforcement learning algorithms.

\subsubsection{MDP}
The warehouse environment can be fully defined using the configurator. Users can specify the warehouse size (number of rows and columns), the locations of packages, and the locations of storage areas. It is also possible to add obstacles to the environment. Additionally, the following hyperparameters can be configured:

\begin{itemize}
    \item Discount factor $\gamma$, where $0.01 \leq \gamma \leq 1$
    
    \item Number of episodes for Q-Learning and SARSA, as well as the number of iterations for Value Iteration and Policy Iteration
\end{itemize}

For both SARSA and online Q-Learning, the following additional hyperparameters can be configured:

\begin{itemize}
    \item $\epsilon$, which defines the $\epsilon$-greedy exploration strategy ($0.01 \leq \epsilon \leq 1$)
    
    \item $\alpha$, the learning rate of the algorithms ($0.01 \leq \alpha \leq 1$)
\end{itemize}

For both Value Iteration and Policy Itertation the warehouse configurator allows to determine $\theta$ which controls when we consider the algorithms to have converged.

\subsubsection{Q-Learning}
In this work, several experiments were conducted using the Q-Learning algorithm in order to evaluate the influence of different hyperparameter settings on the learning performance and convergence behavior.

First, the learning rate $\alpha$ was varied between $\alpha = 0.1$ and $\alpha = 0.01$ to analyze the effect of faster versus more conservative value updates. Second, different exploration strategies were investigated by varying the $\epsilon$ parameter of the $\epsilon$-greedy policy. In addition to a high-exploration setting ($\epsilon = 0.9$) and a moderately high-exploration setting ($\epsilon = 0.75$), a lower exploration baseline ($\epsilon = 0.1$) was included to provide a more exploitation-oriented comparison. The discount factor $\gamma$ was set to 0.99.

Finally, an $\epsilon$-decay strategy was evaluated, starting from an initial value of $\epsilon = 0.9$ and gradually decreasing it over time using a decay factor of $0.99$. The decay was applied until a minimum value of $\epsilon_{\min} = 0.01$ was reached. This approach enables a transition from exploration in the early training phase to exploitation in later episodes. In all Experiments the environment size is a 8 x 8 grid space with two packages and two storages.

\subsection{Discussion}

In the first experiment, both Q-Learning and SARSA exhibited very similar learning behaviour. The convergence speed, cumulative reward development, and overall policy quality were nearly identical during most of the training process.

This similarity can be explained by the chosen hyperparameter configuration. A relatively high initial exploration rate ($\epsilon = 0.9$) caused both algorithms to explore the environment extensively during early episodes. Under strong exploration, the difference between the off-policy update of Q-Learning and the on-policy update of SARSA becomes less pronounced, since both algorithms frequently execute exploratory actions rather than strictly greedy ones.

Additionally, the shared discount factor ($\gamma = 0.99$) and identical learning rate ensured comparable value propagation across states. Because the environment dynamics remained stable during this experiment, both algorithms were able to learn similar value estimates and consequently converged towards comparable policies.

Only minor differences may appear in later stages of learning, where SARSA tends to learn slightly safer policies due to its on-policy nature, while Q-Learning optimizes toward the greedy target policy. However, under the given experimental conditions, these theoretical differences had limited practical impact.

\section{Robustness}

To evaluate the robustness of the reinforcement learning algorithms, multiple runs with different random seeds were conducted for each hyperparameter configuration. This allows assessing the sensitivity of Q-Learning and SARSA to stochastic initialization and exploration behavior.

\subsection{Robustness Analysis}
Robustness is measured in terms of convergence stability, variance in cumulative rewards, success rate of reaching the goal state, and policy change rate over time. For each configuration, multiple episodes were executed under identical environmental settings, while only the random seed was varied.

Additionally, both algorithms were evaluated under the same hyperparameter settings in order to ensure a fair comparison between on-policy (SARSA) and off-policy (Q-Learning) learning behavior.

To further analyze robustness under non-stationary conditions, an additional experiment was introduced in which random obstacles were dynamically added to the environment during the learning process. This simulates unexpected changes in the environment structure and allows evaluating how quickly the agents adapt and recover their performance after such perturbations. The comparison focuses on the resilience of Q-Learning and SARSA with respect to sudden changes in the state space and optimal policy.
